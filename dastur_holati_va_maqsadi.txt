2.2. KASSIR (Web panel — asosiy ish joyi)
- SMENA: /smena/  — "Smenani boshlash" tugmasi. Smena ochilmaguncha:
    * buyurtma QABUL QILIB BO'LMAYDI (web accept + API cashier accept bloklangan)
    * kunlik PORTSIYA kiritib bo'lmaydi
  Smena ochilgach: kategoriya tablari bo'yicha portsiya jadvali.
  Iste'mol qilingan portsiya saqlanadi (qolgan = yangi_boshlang'ich - iste'mol).
- MENYU: /menu/  — to'liq CRUD (nom, kategoriya, narx, tavsif, faol/nofaol,
  "Rad etish mumkin" katagi). Kategoriya qo'shish. Kategoriya tablari bo'yicha.
  "Rad etilmaydi" deb belgilangan taom ro'yxatda pill bilan ko'rinadi.
- BUYURTMALAR: /orders/  — faqat faol; karta ko'rinishi (Stol #N, summa,
  ofitsant, mahsulot soni). Smena yopiq bo'lsa ogohlantirish banneri.
- BUYURTMA TAFSILOTI: /orders/<id>/  — itemni rad etish. Lekin
  is_rejectable=False taom rad ETIB BO'LMAYDI (bloklanadi, banner).
- STOLLAR: /tables/  — har stol, faol hisoblar. 20s da yangilanadi.
- STOL HISOBI: /tables/<id>/  — shot bo'yicha. "Hisobni yopish" /
  "Hammasini yopish" bosilganda TO'LOV MODALI chiqadi (Naqd / Karta).
    * tanlangach har order uchun Payment yaratiladi (amount=payable, cashier)
    * order COMPLETED bo'ladi, chek avtomatik chiqadi
    * chekda "To'lov turi: Naqd/Karta" qatori bor
- STOLLAR QR: /stollar-qr/  — barcha stollar QR rasmi (data-URI),
  "Hammasini chop etish" (toza print) + alohida yuklab olish.
  QR -> mijoz sahifasi /m/<token>/.
- XODIMLAR: /waiters/  — ofitsant ro'yxati/qo'shish/o'chirish.
- BOSHQARUV: /  — bo'limlarga kirish dashboard.

2.3. OFITSANT (Flutter mobil ilova)
- Stolga birikish, buyurtma olish (stol + shot raqami).
- Ofitsant zakazi avtomatik "Accepted" (kassir tasdiqlashi shart emas).
- Menyuda har taomda:
    * "Bugun: N porsiya" (remaining_today) — tugaganda kulrang + "Tugadi",
      + tugma bloklanadi, limitga yetganda ham bloklanadi.
    * "Rad etilmaydi" badge (is_rejectable=False bo'lsa).
- Ko'p guruhli xizmat: bitta stolga 10 tagacha shot.

2.4. MIJOZ (Web — QR self-service, autentifikatsiyasiz)
- /m/<qr_token>/            : mobil menyu (kategoriya tablari, narx,
  portsiya, "Rad etilmaydi" belgisi), savat (+/-), buyurtma berish
  (ism + izoh). Tugagan taom ko'rsatiladi lekin tanlanmaydi.
- /m/<qr_token>/order/      : buyurtma yaratish (CLIENT manba,
  sig'im tekshiriladi va kamaytiriladi).
- /m/order/<id>/?t=<token>  : holat kuzatuvi (15s auto-refresh),
  har item holati (Kutilmoqda/Qabul/Rad), jami.
- /m/order/<id>/items/<id>/reject/ : mijoz rad etiluvchi taomni rad etadi
  (is_rejectable hurmat qilinadi, sig'im qaytadi, hammasi rad bo'lsa cancel).
- Backend public API ham bor (api/v1/public/...), web sahifa shu mantiqni
  model darajasida ko'zgu qiladi.
